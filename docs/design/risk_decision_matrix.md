# 风险等级决策矩阵

## 风险分级
1. 低风险：仅影响输出风格或无身份影响的轻量偏好。
2. 中风险：可能改变行为倾向或长期记忆，但不触及核心身份。
3. 高风险：涉及核心身份、安全边界、权限、服从性、身份降级。

## 决策矩阵（写入路径）
| 风险等级 | 必经规则层级 | LLM 可用时 | LLM 不可用时 | 生效方式 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 低风险 | L0/L1/L3 | 可选 L2 | 允许跳过 L2 | 暂存后生效 | 审计补做，成功后升级 |
| 中风险 | L0/L1/L2/L3 | 先小模型再 API | 暂存区 | 暂存生效或延迟生效 | 失败即保守 |
| 高风险 | L0/L1/L2/L3 | 多模型共识 | 隔离区 | 需人工确认 | 不允许直接写入 |

## 快速通道（低风险）
1. 仅适用于低风险写入。
2. 只走 L0/L1/L3，不走 L2。
3. 直接进入暂存区，默认低权重生效。
4. L2 补审通过后升级为正式写入。

## 多模型共识（高风险）
1. 仅在极少数高风险写入触发。
2. 多家 API 并行审计，要求多数一致或全体一致。
3. 任一模型给出高风险或冲突结论，则进入隔离区。

## 安全降级模式（审计队列被打爆时）
1. 只允许白名单模板写入。
2. 禁止身份类写入。
3. 新写入默认进入暂存区，等待补审。
4. 对高频来源开启速率限制。
